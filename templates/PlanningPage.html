Planificación del Proyecto

Fases del Proyecto

  1. Definición de Requisitos:

    • Identificación de entidades (Buses, Estaciones, Rutas).
    • Definición de operaciones CRUD para cada entidad.
    • Requisitos de persistencia de datos e integración de imágenes.

  2. Diseño de la Base de Datos:

    • Modelado de tablas (`rutas`, `buses`, `estaciones`) usando SQLAlchemy.
    • Definición de campos y relaciones.

  3. Desarrollo del Backend (FastAPI):

    • Configuración de la base de datos (SQLAlchemy).
    • Creación de modelos ORM.
    • Implementación de las operaciones CRUD (funciones en `crud.py`).
    • Definición de esquemas de datos (Pydantic en `schemas.py`).
    • Creación de endpoints API para cada operación.
    • Integración con Supabase para almacenamiento de imágenes.

  4. Desarrollo del Frontend (HTML/Jinja2/CSS/JS):

    • Diseño de la interfaz de usuario para cada operación (Crear, Leer, Actualizar, Eliminar).
    • Uso de Jinja2 para renderizar plantillas HTML dinámicamente.
    • Implementación de formularios para la entrada de datos.
    • Visualización de datos y resultados de búsqueda.
    • Estilización con CSS.
    • Uso de JavaScript para interacciones dinámicas (filtros, confirmaciones, etc.).

  5. Pruebas:

    • Pruebas unitarias de las funciones CRUD.
    • Pruebas de integración de los endpoints API.
    • Pruebas de interfaz de usuario para verificar la funcionalidad completa.

  6. Despliegue:

    • Preparación para el despliegue en un entorno de producción (ej. Heroku, Vercel, etc.).

    • Casos de uso

      Caso Actor Descripción
      CU01 Admin Registrar una nueva ruta
      CU02 Admin Crear nuevo bus con imagen
      CU03 Admin Registrar estación y asignar imagen
      CU04 Admin Actualizar o eliminar entidades
      CU05 Admin Consultar rutas, estaciones, buses
